Job generation

Problem: Exploration of web and generation of jobs.

Goals: Supports

  • Different strategies for exploring the web.
  • Different representation of jobs.
  • Simple pipeline without web exploration eg downloader, disk scanning parser.

Approach

Explicit, centralized representation of task in system:

  • Centralized logic for generation of jobs, directing crawl.
  • Strategy pattern.

Rationale

The benefits are:

  • Exchangeable crawling strategy for different application scenarios, strategies for web exploration.
  • By being the central point for job generation, all URLs pass here through.
    This allows, for instance, for system-wide URL checking.
