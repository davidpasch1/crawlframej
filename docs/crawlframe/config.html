Configuration

The configuration of the framework is done by

  • setting the parameters of the stages of the pipeline and
  • through the CrawlerConfig class which stores the global settings.

Important: The CrawlerConfig class has to be initialized before the framework can be used.

Media Types

The framework needs to know the type of the fetched data for the following purposes.

  • Stage acceptance: Definition of an acceptance criterion for the data for processing by a particular stage.
  • Normalized file extensions: Normalization of file extensions of the output files.

The problem is that the content's type is represented differently in the web and on the file system. For this purpose, Linux has a built-in mapping which translates between the two representations. The mapping is stored in the following files.

/etc/mime.types	# global MIME types file
~/.mime.types		# optional user MIME types file: overrides the global one

Note: The mapping is a M:N relation between media types and file extensions.

The stage acceptance criterion for the fetched data is based on it's media type or file extension eg the webpage parser accepts text/html or [.html, .htm].

Normalization of file extensions

The file extension of an output file is normalized by appending or replacing it's file extension by the default file extension for the media type eg text/html is written as .html.

Note: This feature has to be explicitly enabled when configuring the framework.

For the default file extension the framework takes the first file extension in the MIME types file for the media type. By creating a local user MIME types file the user can change the default order of the file extensions in the global MIME types file.
